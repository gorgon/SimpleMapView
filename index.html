Welcome to SimpleMapView project

It can help you to understand some ideas about managing a huge remote field/map which could be divided into tiles.

Main ideas:

  • 2 cache levels:
    • RAM cache
    • Disc cache
  • Disc cache access (save/read) and remote requests are handled asynchronously
  • To prevent handling tiles for fast scrolled screens
    • 2 request queues are used inside asynchronous handler
  • To prevent OutOfMemory error
    • RAM bitmaps are stored as a limited pull and re-used after RAM size reaches its maximum (could be configured) size
